# Code audit snapshot — RBAC & roles

**Date:** 2026-05-16

## Prisma

- `UserRole`: STUDENT, TEACHER, ADMIN, SUPER_ADMIN — `schema.prisma`
- `UserAccountStatus`: ACTIVE, PAUSED, LEAVED, BLOCKED — not checked at login (gap)

## API enforcement

- Guards: `auth.guard.ts`, `gql-auth.guard.ts` — JWT/cookie → `user.id` only
- Admin: `auth.ts` `requireAdmin()`, `createUserAsAdmin()`, REST `/admin/users`, GraphQL `AdminResolver`
- Students list: `users.service.ts` `listStudents` — TEACHER scoped by `teacherId`
- Lessons: `be-lessons.ts` — membership on read/update/delete; create unchecked (gap)
- Vocabulary: optional `studentId` without staff check (gap)

## Web UI

- Matrix: `apps/web/src/mocks/roles.ts` — canView/edit/manage/schedule per feature
- Nav: `Sidebar.tsx` — `/students` teacher+; `/admin` admin+
- No `middleware.ts` route protection

## SUPER_ADMIN CLI

- `scripts/super-admin.ts` — token, create, revoke, delete, list
- `npm run super-admin`
